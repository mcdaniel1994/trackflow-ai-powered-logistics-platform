# `memory-bank/`

Persistent TrackFlow project context for coding agents.

Read these files at the start of every session before planning or editing:

- `projectbrief.md` - business context, objectives, operational problems, and stakeholders
- `techContext.md` - stack, architecture, repository boundaries, and current technical decisions
- `progress.md` - delivered engagements, active work, planned roadmap, and known risks

The memory bank is TrackFlow-specific. Keep it concrete and update it when an engagement changes the platform's architecture, delivered status, or roadmap.
